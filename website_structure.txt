[2. FOLDER RESPONSIBILITIES]
--------------------------------------------------------------------------------
- ROOT (*.html): One file per page, flat structure. Intentional - each file
  maps 1:1 to a future WordPress page-template. Never nest pages in
  sub-folders.
- assets/css: All stylesheets. Three tiers only - GLOBAL (site-wide
  singletons), COMPONENT (shared by 2+ pages, named after the component),
  PAGE (used by exactly 1 page, named after that page). See rules.txt
  Section 1 for the authoritative rule; this file only maps current files
  to those tiers.
- assets/js: Same three-tier logic as CSS. main.js is the ONLY place global
  interactive behavior (navbar, mega menu, language switcher, search
  overlay) may live - never duplicate this logic in a page file.
- assets/images: Categorized sub-folders only (logo, portfolio,
  clients-logo, ...). No loose files directly in assets/images root. No
  unsorted/raw folders (see known gap below) - every folder here is
  considered PRODUCTION-READY, optimized, and referenced by the codebase.
- Media/: Source/marketing video, not production assets. Treat as archival
  until a compression + poster-image + lazy-load strategy is applied
  (rules.txt Section 8).
- Root *.txt files: documentation and process only. Never code.

[7. CSS ARCHITECTURE SUMMARY]
--------------------------------------------------------------------------------
Three-tier cascade, loaded in this order on every page:
  1. GLOBAL tokens + navbar.css + footer.css + page-hero.css
  2. COMPONENT css (ceo_section.css, clients_marquee.css) - only on pages
     that use that component
  3. PAGE css (e.g. services.css) - loaded last so it can override
     component-level spacing/variants without !important
Naming, prefixing, token, and spacing rules are fully defined in rules.txt
Sections 2-3 and are NOT repeated here to avoid two sources of truth.

[9. FUTURE WORDPRESS MAPPING (SUMMARY)]
--------------------------------------------------------------------------------
Full detail lives in wordpress-roadmap.txt. Quick reference only:
  - Root *.html files          -> page-*.php template files
  - navbar.css/footer.css/     -> theme-wide enqueued styles (style.css or
    page-hero.css                a compiled global.css)
  - Page CSS/JS                -> conditionally enqueued per-template assets
  - Component CSS/JS           -> block/template-part specific assets
  - [TEMPLATE: header.php] /
    [TEMPLATE: footer.php]     -> header.php / footer.php
  - [TEMPLATE PART: x.php]     -> template-parts/x.php (or Gutenberg block)
  - Industries list, mega-menu structure, testimonials, clients marquee,
    stats -> CPT / ACF Repeater candidates
No WordPress conversion work begins from this document. This section
exists purely so nothing built today creates rework later.
