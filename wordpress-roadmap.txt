[3. ASSET ENQUEUE STRATEGY]
--------------------------------------------------------------------------------
- GLOBAL CSS (navbar.css, footer.css, page-hero.css, future tokens.css)
  -> enqueued unconditionally in functions.php on every template.
- COMPONENT CSS (ceo_section.css, clients_marquee.css, future
  insights_section.css) -> enqueued conditionally, only on templates/
  template-parts that actually output that component
  (`is_page_template()` / `has_block()` checks).
- PAGE CSS (index.css, services.css, ...) -> enqueued conditionally per
  template, one enqueue call per page template.
- JS follows the identical three-tier logic: main.js always loads;
  component JS loads with its component; page JS loads with its page
  template.
- Design tokens: once tokens.css exists (rules.txt Section 3/16 debt item)
  it becomes the FIRST global stylesheet enqueued, before navbar.css, so
  every other stylesheet can rely on the custom properties being defined.

[5. MENUS & NAVIGATION]
--------------------------------------------------------------------------------
Full menu content lives in navigation-map.txt. Mapping:
  - Top bar contact/social/language -> Theme Options (ACF Options Page) so
    phone/email/social links are editable without touching template code.
  - Main mega menu -> WordPress Custom Menu with a mega-menu walker class,
    OR an ACF Flexible Content "Mega Menu Column" repeater if the
    structure needs to stay editor-manageable.
  - Industries list (26 items) -> strong CPT/Taxonomy candidate (Section
    6) rather than a hardcoded menu, since industries are likely to be
    referenced from multiple pages (industry landing pages, filters on
    portfolio/events).

--------------------------------------------------------------------------------
  Content block                | CPT / Taxonomy candidate
  -------------------------------|----------------------------------------
  Industries list (26 items)     | Taxonomy: `industry` (attached to

  Events (Owned/Co-Organized/     | CPT: `event`, with taxonomy

  Conferences/Forums)             |

  Testimonials                    | CPT: `testimonial`
  Clients / partner logos         | CPT: `client` (or ACF repeater if the
                                  | list stays simple - decide based on
                                  | whether clients ever need individual

Decision on CPT vs ACF-repeater-on-a-page for each of the above should be
made when content volume and edit-frequency are known - not guessed now.

--------------------------------------------------------------------------------
[8. MEDIA HANDLING]
--------------------------------------------------------------------------------
- Media/*.mp4 marketing videos -> WP Media Library only after compression
  and poster-image generation (rules.txt Section 8); large video should
  seriously be considered for external hosting (YouTube/Vimeo embed or a
  CDN) rather than self-hosted in wp-content/uploads, for performance.
- assets/images/* -> migrates into the WP Media Library on a per-template
  basis as templates are built, keeping the same category structure
  (logo, portfolio, clients-logo) as folder-equivalent tags/categories if
  the media library is organized with a plugin, otherwise flat with
  descriptive filenames (already required by rules.txt Section 2).
- The unsorted assets/images/Images Website Badarexpo/ folder (see
  website_structure.txt Section 6, Known Gap #2) must be resolved -
  cleaned, renamed, and either discarded or properly categorized - BEFORE
  any media migration begins. Do not migrate raw export dumps into the
  Media Library.
